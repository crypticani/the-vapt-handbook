# 07 — CTF Writeups: Methodology

> How to systematically approach any CTF challenge and document your solution.

## 🔴 The Framework

```
STEP 1: UNDERSTAND (2 min)
→ Read the challenge description carefully. What's the goal?

STEP 2: ENUMERATE (5 min)
→ Gather all available information. Don't attack yet.

STEP 3: HYPOTHESIZE (2 min)
→ What's the likely vulnerability? What's the intended path?

STEP 4: TEST (varies)
→ Validate your hypothesis. Modify and retry.

STEP 5: EXPLOIT (varies)
→ Craft the exploit. Capture the flag.

STEP 6: DOCUMENT (5 min)
→ Write the writeup while it's fresh.
```

## 🔴 If You're Stuck

```
1. Re-read the challenge description — there's usually a hint
2. Check the page source / file metadata — flags hide in comments
3. Google the exact error message
4. Try the simplest possible attack first
5. Look for similar writeups from past CTFs
6. Take a break and come back
7. Ask for hints in the CTF Discord (if allowed)
```
